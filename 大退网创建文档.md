# 大退网 — 创建全流程文档

## 一、项目概述

**大退网**（DaTuiWang）是一个中国演唱会票务退款投诉数据可视化平台，以深色主题 + 红橙强调色的数据新闻风格，通过图表、表格、案例卡片等形式，呈现演唱会退票领域的消费者投诉全景。

- **技术栈**：纯静态单页 HTML + CSS + JavaScript，使用 Chart.js 绘制图表，Google Fonts 加载中文字体
- **部署方式**：Gitee 代码托管 + Netlify 静态网站托管
- **在线地址**：https://beamish-stroopwafel-6396c7.netlify.app

## 二、网站创建步骤

### 第 1 步：数据采集

通过以下公开渠道采集演唱会退票投诉相关数据：

| 数据来源 | 采集方式 | 采集内容 |
|---------|---------|---------|
| 全国 12315 平台 | 浏览器访问 https://tsgs.12315.cn ，查看首页公开统计数据 | 全国投诉调解成功率、热点问题排行 |
| 黑猫投诉 | 网络搜索 + 抓取《2025年度娱乐票务领域投诉数据报告》 | 娱乐票务投诉总量、退款困难占比、平台回复率、年龄段分布 |

### 第 3 步：网站设计与开发

使用前端设计技能，生成一个完整的单页 HTML 文件（`index.html`，共约 1,276 行），包含：

- **CSS**：深色主题（`#0a0a0a` 背景 + `#ff4444`/`#ff6b35` 红橙强调色），响应式布局，滚动动画
- **JavaScript**：Chart.js 图表渲染、数字计数器动画、滚动揭示动画、导航交互
- **字体**：Google Fonts Noto Sans SC
- **图表**：3 个 Chart.js 交互式图表（平台投诉量柱状图、投诉类型环形图、涉诉金额柱状图）

### 第 4 步：代码托管与部署

1. **Gitee 代码托管**：初始化 Git 仓库，推送到 `https://gitee.com/leox24/datui`
2. **Netlify 静态部署**：通过 Netlify CLI 匿名部署，获得在线访问地址

## 四、数据每日更新方案

当前网站是纯静态 HTML，数据以硬编码方式写入页面。要实现数据每日自动更新，有以下几种方案：

### 方案 A：GitHub Actions + Python 自动抓取（推荐）

**原理**：每天定时运行 Python 脚本，从各数据源抓取最新数据，自动更新 HTML 文件并部署。

**步骤**：

1. **编写 Python 抓取脚本** `update_data.py`：
   - 用 `requests` + `BeautifulSoup` 抓取黑猫投诉、消费保等平台的公开数据
   - 用正则或 JSON 解析提取关键数字
   - 将新数据写入 `data.json`

2. **改造 HTML 为模板驱动**：
   - 将 `index.html` 中的硬编码数据替换为 JavaScript 变量
   - 页面加载时从 `data.json` 读取数据并渲染图表

3. **配置 GitHub Actions**（`.github/workflows/daily-update.yml`）：
   ```yaml
   name: Daily Data Update
   on:
     schedule:
       - cron: '0 2 * * *'  # 每天北京时间 10:00 执行
   jobs:
     update:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with:
             python-version: '3.11'
         - run: pip install requests beautifulsoup4
         - run: python update_data.py
         - run: git add . && git commit -m "data: 每日数据更新" && git push
   ```

**优点**：全自动、免费、可追溯历史版本
**缺点**：需要将代码迁移到 GitHub（Gitee Actions 功能有限）

### 方案 B：前端 JavaScript 实时拉取

**原理**：页面加载时，通过 JavaScript 从各平台公开 API 或网页实时获取数据。

**步骤**：

1. 编写 `fetch_data.js`，使用 `fetch()` 或 CORS 代理请求各数据源
2. 将抓取结果动态填充到页面 DOM 中
3. 设置缓存策略（如 localStorage），避免每次刷新都请求

```javascript
// 示例：页面加载时获取最新数据
async function loadLatestData() {
  const response = await fetch('https://your-api-proxy.com/api/datui');
  const data = await response.json();
  updateCharts(data);
  updateCounters(data);
}
window.addEventListener('DOMContentLoaded', loadLatestData);
```

**优点**：每次访问都是最新数据
**缺点**：依赖 CORS 代理；各平台可能随时变更页面结构导致抓取失败

### 方案 C：自建后端 API + 定时任务

**原理**：搭建一个简单的后端服务（如 Python Flask / Node.js），定时抓取数据并提供 API。

**步骤**：

1. 部署一个云服务器（如腾讯云轻量服务器，约 50 元/月）
2. 编写后端服务，包含：
   - 定时任务（cron）：每天抓取各平台数据，存入数据库
   - REST API：`/api/stats` 返回最新统计数据
3. 前端页面通过 `fetch('/api/stats')` 获取数据

**优点**：最灵活、最稳定
**缺点**：需要服务器成本、需要维护

### 方案对比

| 维度 | 方案 A (GitHub Actions) | 方案 B (前端实时) | 方案 C (自建后端) |
|------|:-:|:-:|:-:|
| 成本 | 免费 | 免费 | ~50 元/月 |
| 技术难度 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 数据实时性 | 每天更新 | 每次访问更新 | 每天更新 |
| 稳定性 | 高 | 中（依赖外部页面结构） | 高 |
| 维护成本 | 低 | 高 | 中 |
| 推荐度 | ★★★★★ | ★★★ | ★★★★ |

### 建议

对于当前阶段，推荐 **方案 A（GitHub Actions + Python 自动抓取）**：

1. 将代码从 Gitee 镜像到 GitHub
2. 编写 Python 抓取脚本，覆盖黑猫投诉、消费保、12315 首页三个数据源
3. 配置 GitHub Actions 每天自动运行
4. 通过 Netlify 自动部署（Netlify 支持 GitHub 集成，push 后自动更新网站）

如果后续数据源增多或需要更复杂的处理，可以升级到方案 C。
